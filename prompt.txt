Why IFERROR?
The outer IFERROR(..., "") wrapper is a safety net:

If the referenced row doesn't exist (e.g., CW_Data has fewer rows)
If there's a data type mismatch
If any unexpected error occurs
→ The formula gracefully returns blank instead of showing an ugly #REF! or #VALUE! error.

Why the "Sparse Display" Approach?
This design means:

✅ Every row in Details has a formula
✅ Only rows where Status = "Delay" show data
✅ On-Time rows appear as blank lines
✅ When you add new data to CW_Data and refresh, delays automatically appear
Aging Bucket (Column J) — Special Formula
This is the only calculated field that doesn't pull directly from CW_Data:

=IFERROR(IF(CW_Data!N{row}="Delay",
  IF(CW_Data!O{row}>60, "60+ Days",
    IF(CW_Data!O{row}>30, "31-60 Days",
      IF(CW_Data!O{row}>14, "15-30 Days",
        IF(CW_Data!O{row}>7, "8-14 Days", "1-7 Days")))), ""), "")
In plain English: "If this is a delayed item, categorize it into an aging bucket based on how many days late it is: 60+, 31-60, 15-30, 8-14, or 1-7 days."

Key Implementation Notes
Sparse Display Pattern: The original Excel uses a "sparse" display where every row has a formula, but only rows with Status="Delay" show data. In React/Python, you should filter first rather than showing blank rows.

Row Offset: In Excel, Details row 64 maps to CW_Data row 59 (offset of 5 rows due to header structure). In code, use zero-indexed arrays after filtering.

## 🔑 Key Formula Functions Explained

| Function | What It Does | Example |
|----------|--------------|---------|
| **COUNTA** | Counts cells that aren't empty | `COUNTA(B4:B100)` = How many suppliers listed |
| **COUNTIF** | Counts cells matching ONE condition | `COUNTIF(J:J,"Delay")` = How many delays |
| **COUNTIFS** | Counts cells matching MULTIPLE conditions | `COUNTIFS(B:B,"PMI",J:J,"Delay")` = PMI delays only |
| **MAX** | Finds the largest number | `MAX(O:O)` = Worst (longest) delay |
| **IFERROR** | Shows alternative value if formula errors | `IFERROR(A1/B1,0)` = Divide, or show 0 if error |
| **IF** | Makes a yes/no decision | `IF(A1>B1,"Bigger","Smaller")` |
| **TEXT** | Formats numbers as text | `TEXT(0.156,"0.0%")` = "15.6%" |
| **ABS** | Removes negative sign | `ABS(-33)` = 33 |
| **& (concatenate)** | Joins text together | `"Total: "&A1` = "Total: 157" |

Top Part (Numerator)	COUNTIFS(LW_Data!I4:I985,"Shipped", LW_Data!J4:J985,"On time")	"Count how many rows have BOTH conditions true:"
1. The Stage column (I) says "Shipped" AND

In other words: "How many items were shipped AND arrived on time?"
Bottom Part (Denominator)	COUNTIFS(LW_Data!I4:I985,"Shipped")	"Count how many rows have Stage = Shipped"

In other words: "How many items were shipped in total (regardless of whether they were on time or late)?"
Division ( / )	Top ÷ Bottom	"Divide on-time shipments by total shipments"

This gives you the On-Time Delivery rate as a decimal (e.g., 0.838 = 83.8%)
Real-World Analogy
Imagine you're a shipping manager checking your team's performance:

🚚 "Out of all the packages we shipped last week, what percentage arrived on time?"

Step 1: Count packages that were shipped AND on time = 100 packages
Step 2: Count ALL packages that were shipped = 119 packages
Step 3: Divide: 100 ÷ 119 = 84% on-time delivery rate
Why Use COUNTIFS Twice?
Function	Purpose
First COUNTIFS	Counts only the "winners" — items that passed BOTH tests (shipped + on time)
Second COUNTIFS	Counts the total pool — all shipped items (the baseline to compare against)
